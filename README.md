# Поиск статей помощи по запросу пользователя (тестовое Авито)

По тексту обращения нужно вернуть топ-10 статей базы помощи (793 статьи). Метрика MAP@10.

Итог: **0.538 на калибровке, 0.531 на тесте**.

Важная оговорка про железо: всё считалось на MacBook M2 с 8 ГБ RAM, и это стало реальным потолком решения. Сильные cross-encoder реранкеры (bge-reranker-v2-m3 и подобные), которые обычно дают основной прирост поверх ретривера, на 8 ГБ запустить не удалось чисто технически: fp32 уходит в swap и вешает систему, int8-квантизация на arm64 без ускорения (0.4 пары/сек), CoreML падает с ошибкой. При этом recall@30 = 0.92, то есть правильные статьи почти всегда есть в кандидатах, их нужно только переранжировать. С 16+ ГБ RAM тот же пайплайн плюс полноценный реранкер реалистично дал бы ~0.55-0.58.

## Запуск

Весь пайплайн собран в ноутбуке `solution.ipynb` (подготовка, скоринг, валидация на калибровке, генерация answer.csv). То же самое можно запустить скриптами:

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
bash scripts/download_models.sh   # веса моделей с HuggingFace, ~4 ГБ

PYTHONPATH=solution .venv/bin/python solution/prep.py
PYTHONPATH=solution .venv/bin/python solution/colbert.py encode_docs
CB_MAXLEN=384 PYTHONPATH=solution .venv/bin/python solution/colbert.py encode_docs
PYTHONPATH=solution .venv/bin/python solution/colbert.py encode_test
PYTHONPATH=solution .venv/bin/python solution/berta.py docs
PYTHONPATH=solution .venv/bin/python solution/berta.py test
PYTHONPATH=solution .venv/bin/python solution/ce_test.py
PYTHONPATH=solution .venv/bin/python solution/gen_answer.py   # -> solution/answer.csv
```

Пайплайн детерминирован, повторный запуск выдаёт тот же answer.csv, что загружен на платформу (проверено побайтово). Пиковая память < 3 ГБ.

## Обработка HTML

`common.py`: вырезаю script/style, блочные теги (`<br>`, `</p>`, `</li>`...) заменяю переводами строк, остальные теги удаляю, декодирую сущности (`&nbsp;` и числовые), схлопываю пробелы. Из статьи беру title + чистый текст.

Тело статьи режу на чанки ~1100 символов с перекрытием 200 (до 8 на статью), к каждому чанку приклеиваю заголовок, иначе у длинных статей релевантный кусок теряется. Из запросов регэкспом убираю приветствия («здравствуйте, подскажите пожалуйста...»), они зашумляют эмбеддинги коротких текстов.

## Модель

Скор статьи = взвешенная сумма пяти сигналов (веса подобраны на калибровке):

```
score = E5 + 0.3*BM25 + 0.7*ColBERT + 0.15*CE + 0.8*BERTA
```

- **E5**: intfloat/multilingual-e5-large, косинус запрос-чанк, по статье среднее top-2 чанков;
- **BM25**: rank_bm25 по леммам (pymorphy3), без стоп-слов;
- **ColBERT**: BAAI/bge-m3 late interaction, вектор на каждый токен, maxsim по чанкам топ-100 кандидатов. Токен-векторы считаются один раз и лежат на диске в fp16 (mmap), поэтому влезает в память. Это замена полноценному реранкеру в условиях 8 ГБ;
- **CE**: cross-encoder/mmarco-mMiniLMv2-L12-H384-v1 (117M), единственный cross-encoder, который влез в память; скорит топ-20 кандидатов;
- **BERTA**: sergeyzh/BERTA, русскоязычный энкодер другого семейства, добавляет разнообразие.

Все модели открытые, < 1B параметров, работают локально без внешних API.

Прирост по шагам (MAP@10, калибровка): BM25 один 0.32, с e5-large 0.50, с ColBERT 0.52, с CE 0.53, с BERTA 0.54.

## Валидация на calibration.f

Метрика MAP@10 в `common.py`, считалась после каждого изменения. На калибровке подбирались: параметры чанкинга, BM25 (k1, b), пулинг чанков, веса фьюжна (грид). Перенос на тест стабильный: 0.538 на калибровке, 0.531 на тесте.

Отдельно: пробовал использовать разметку калибровки как обучающие данные (kNN по похожим запросам + градиентный бустинг). На честной 5-fold CV внутри калибровки получалось 0.885, но на тесте только 0.448: внутри калибровки много почти одинаковых запросов, а с тестом пересечений нет, и kNN вне распределения только вредит. Откатился на ансамбль без обучения на разметке. Вывод зафиксирован как урок: CV не ловит сдвиг распределения между выборками.

## Анализ ошибок

Смотрел AP по запросам калибровки на финальном ансамбле: 65/500 полных промахов, 152/500 идеальных.

1. Главная ошибка: путаница соседних статей одной темы, на вопрос о стоимости доставки прилетает «Оплата заказов с доставкой» вместо «Кто оплачивает доставку и сколько она стоит». Частично закрыто ColBERT + CE (вместе +0.03-0.04). Остаток как раз то, что добил бы крупный реранкер на 16 ГБ.
2. Разговорные формулировки («не работает за наличку??») плохо матчатся и лексикой, и эмбеддингами. Помогает ансамбль разных семейств моделей.
3. Опечатки («Москлвской»): BM25 по леммам их не прощает, вытягивают субтокенные модели.
4. Шум: приветствия (вырезаю), маскировка `<MONEY>` (оставил, она одинакова в калибровке и тесте), обрывки диалогов.
5. Длинные статьи: решено чанкингом (+0.02 против простого усечения).

Что ещё пробовал и выкинул: char-TFIDF (вытеснен ColBERT), RRF-фьюжн (хуже суммы), PRF/Rocchio (хуже базы), bge-reranker-base через ONNX (портит ранжирование), bge-m3 dense и USER2-base (коррелируют с имеющимися сигналами, вес 0 в гриде).
